# Client Billing (Running Account - RA Bills) - Indian Construction Standard

**Date**: February 4, 2026  
**Topic**: How Project Billing to Client Works

---

## 🎯 **WHAT IS CLIENT BILLING?**

**Client Billing** (also called **RA Bills** - Running Account Bills or **Progress Bills**) is how construction companies invoice their clients based on work completed.

---

## 📊 **CLIENT BILLING FLOW**

```
Project BOQ (Contract with Client)
    ↓
Work Completed (Measured on Site)
    ↓
RA Bill Preparation (Monthly/Milestone-based)
    ↓
Client Verification & Approval
    ↓
Payment from Client
    ↓
TDS Deduction by Client
    ↓
Net Amount Received
```

## 🔄 **BILLING CYCLE**

### **Monthly Billing (Most Common)**

```
Month 1 (Jan):
- Work Done: ₹10,00,000
- RA Bill #1: ₹10,00,000
- Retention (10%): ₹1,00,000
- TDS (2%): ₹20,000
- Net Received: ₹8,80,000

Month 2 (Feb):
- Work Done: ₹15,00,000
- RA Bill #2: ₹15,00,000
- Cumulative: ₹25,00,000
- Retention (10%): ₹1,50,000
- TDS (2%): ₹30,000
- Net Received: ₹13,20,000

Month 3 (Mar):
- Work Done: ₹6,87,500
- RA Bill #3: ₹6,87,500
- Cumulative: ₹31,87,500
- Retention (10%): ₹68,750
- TDS (2%): ₹13,750
- Net Received: ₹6,05,000
```

### **Milestone-Based Billing**

```
Milestone 1: Excavation Complete (20%)
- Contract Value: ₹50,00,000
- Milestone Amount: ₹10,00,000
- Bill #1: ₹10,00,000

Milestone 2: D-Wall Complete (50%)
- Milestone Amount: ₹25,00,000
- Less: Previous Bill: ₹10,00,000
- Bill #2: ₹15,00,000

Milestone 3: Backfilling Complete (80%)
- Milestone Amount: ₹40,00,000
- Less: Previous Bills: ₹25,00,000
- Bill #3: ₹15,00,000
```

## 💰 **DEDUCTIONS IN CLIENT BILLING**

### **1. Retention Money (Security Deposit)**
```
Purpose: Ensure defect rectification
Typical: 5% to 10% of bill amount
Release: On project completion + defect liability period
Duration: Usually 6-12 months after completion

Example:
Bill Amount: ₹10,00,000
Retention @ 10%: ₹1,00,000
Net After Retention: ₹9,00,000
```

### **2. TDS (Tax Deducted at Source)**
```
Section: 194C (Contractor Payments)
Rate: 1% (if PAN available) or 2% (if TDS certificate)
Deducted by: Client
Refundable: Yes, when filing IT returns

Example:
Bill Amount: ₹10,00,000
TDS @ 2%: ₹20,000
Net After TDS: ₹9,80,000
```

### **3. GST**
```
GST Rate: 18% (construction services)
Collected by: Contractor (you)
Paid to: Government

Example:
Work Value: ₹10,00,000
GST @ 18%: ₹1,80,000
Total Bill: ₹11,80,000

Client Pays: ₹11,80,000
You Pay Govt: ₹1,80,000 (minus input credit)
```

### **4. Advance Adjustment**
```
If client gave advance at project start:
Advance Received: ₹5,00,000

Bill #1: ₹10,00,000
Less: Advance Adjustment: ₹5,00,000
Net Payable: ₹5,00,000
```

## 🔧 **DATABASE SCHEMA - CLIENT BILLING**

```sql
CREATE TABLE client_bills (
    id INT PRIMARY KEY AUTO_INCREMENT,
    bill_number VARCHAR(50) UNIQUE NOT NULL,  -- RA/2026/001
    bill_type ENUM('running_account', 'milestone', 'final', 'supplementary') DEFAULT 'running_account',
    bill_date DATE NOT NULL,
    
    -- References
    project_id INT NOT NULL,
    client_id INT NOT NULL,
    
    -- Period
    period_from DATE,
    period_to DATE,
    milestone_id INT,  -- If milestone-based
    
    -- Amounts
    gross_amount DECIMAL(15,2),
    previous_bills_amount DECIMAL(15,2),
    cumulative_amount DECIMAL(15,2),
    
    -- Deductions
    retention_percentage DECIMAL(5,2) DEFAULT 10.00,
    retention_amount DECIMAL(15,2),
    tds_percentage DECIMAL(5,2) DEFAULT 2.00,
    tds_amount DECIMAL(15,2),
    advance_adjustment DECIMAL(15,2) DEFAULT 0,
    other_deductions DECIMAL(15,2) DEFAULT 0,
    
    -- GST
    gst_percentage DECIMAL(5,2) DEFAULT 18.00,
    gst_amount DECIMAL(15,2),
    
    -- Net Amount
    net_amount DECIMAL(15,2),
    
    -- Status
    status ENUM('draft', 'submitted', 'client_verified', 'approved', 'paid', 'rejected'),
    
    -- Verification
    prepared_by INT,
    verified_by INT,
    approved_by INT,
    submitted_at DATETIME,
    approved_at DATETIME,
    
    -- Payment
    payment_received_date DATE,
    payment_reference VARCHAR(100),
    
    -- Remarks
    remarks TEXT,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (client_id) REFERENCES clients(id)
);

CREATE TABLE client_bill_items (
    id INT PRIMARY KEY AUTO_INCREMENT,
    bill_id INT NOT NULL,
    
    -- Work Reference
    boq_item_id INT,
    work_item_type_id INT,
    description VARCHAR(500),
    
    -- Quantities
    boq_quantity DECIMAL(15,3),
    previous_billed_quantity DECIMAL(15,3),
    current_quantity DECIMAL(15,3),
    cumulative_quantity DECIMAL(15,3),
    balance_quantity DECIMAL(15,3),
    
    -- Rates & Amounts
    unit VARCHAR(50),
    rate DECIMAL(15,2),
    amount DECIMAL(15,2),
    
    -- Verification
    verified_quantity DECIMAL(15,3),  -- Client may verify different qty
    verified_amount DECIMAL(15,2),
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (bill_id) REFERENCES client_bills(id) ON DELETE CASCADE,
    FOREIGN KEY (boq_item_id) REFERENCES project_boq_items(id),
    FOREIGN KEY (work_item_type_id) REFERENCES work_item_types(id)
);

-- Retention Money Tracking
CREATE TABLE retention_money (
    id INT PRIMARY KEY AUTO_INCREMENT,
    project_id INT NOT NULL,
    client_id INT NOT NULL,
    client_bill_id INT,
    
    retention_amount DECIMAL(15,2),
    retention_percentage DECIMAL(5,2),
    
    status ENUM('held', 'released', 'partially_released'),
    
    release_date DATE,
    release_amount DECIMAL(15,2),
    release_reference VARCHAR(100),
    release_remarks TEXT,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (client_id) REFERENCES clients(id),
    FOREIGN KEY (client_bill_id) REFERENCES client_bills(id)
);
```

## ✅ **IMPLEMENTATION PRIORITY**

### **Phase 1: Contractor Billing** (4 weeks) - FIRST
- Prevents overpayment to contractors
- Immediate cost savings

### **Phase 2: Client Billing** (3 weeks) - SECOND
- Ensures timely invoicing to clients
- Improves cash flow

### **Phase 3: HR & Payroll** (6 weeks) - THIRD
- Employee management
- Statutory compliance

---

**Summary**: 
- **Contractor Billing** = Your COST (what you pay)
- **Client Billing** = Your INCOME (what you receive)
- **Difference** = Your PROFIT

Both are critical for business health! 💰
